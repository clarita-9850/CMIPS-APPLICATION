1 Person, Case & State Hearing Search

‚ñº

CMIPS provides comprehensive search capabilities to find persons (recipients, providers, applicants), cases, and state hearings.

Person Search

‚ñº

Purpose:

Search for existing persons in CMIPS before creating new person records to avoid duplicates.

Search Criteria:

Field Description
First Name Person's first name (partial match supported)
Last Name Person's last name (partial match supported)
SSN Social Security Number (full or last 4 digits)
Date of Birth Person's birth date
CIN Client Index Number from SCI
Person ID CMIPS internal person identifier
Person Type Recipient, Provider, Applicant, Referral
County County where person is associated

Search Results:

Results display:

  • Person name, SSN (masked: XXX-XX-1234), date of birth
  • Person type (Recipient, Provider, both)
  • Active cases
  • County association
  • Quick access to Person Home screen

Duplicate Prevention

ALWAYS search before creating new person!

  • Search by SSN and date of birth
  • Check for similar names with different SSNs
  • Review existing person details before deciding to create new
  • Creating duplicate persons causes data integrity issues

Advanced Search Features:

  • Wildcard Search: Use * for partial name matching (e.g., "John*" finds John, Johnny, Johnathan)
  • Soundex Search: Finds phonetically similar names
  • Multi-Criteria: Combine multiple search fields to narrow results
  • County Filter: Limit search to specific county or search statewide

‚ñº

  • Case Number: Associated IHSS case

  • Hearing Status: Pending, Scheduled, Completed, Withdrawn
  • Filed Date Range: Date range when appeal filed

Scheduled Hearing date and time set
Postponed Hearing rescheduled

Aid Paid Pending

When recipient appeals adverse action (termination, reduction), services may continue pending hearing decision if:

  • Appeal filed within 10 days of NOA
  • Recipient requests continuation of services
  • "Aid Paid Pending" status set on case

Intake Worker Actions:

Step 1: Gather Basic Information

  • Person's name, date of birth
  • Contact information (phone, address)
  • Social Security Number (if available)
  • Nature of inquiry (seeking services, question, complaint)

Step 2: Search for Existing Record

  • Check if person already in system
  • Review any existing cases or applications

Step 3: Determine Next Steps

  • If existing case: Answer questions, address concerns
  • If existing application: Check status, provide update
  • If new inquiry: Create referral or application

Step 4: Provide Information

  • Explain IHSS program eligibility
  • Describe application process
  • Identify required documentation
  • Set expectations for timeline

Referral vs. Application

‚ñº

Referral:

A referral is an initial inquiry or contact that may or may not lead to an application.

When to Create Referral:

  • Person is inquiring about program eligibility
  • Person is considering applying but hasn't decided
  • External agency refers person for possible services
  • Person needs more information before applying
  • Follow-up needed to determine if person wants to proceed

Application:

An application is a formal request for IHSS services.

When to Create Application:

  • Person explicitly requests to apply for IHSS
  • Person has completed application form (SOC 873)
  • Person appears to meet basic eligibility criteria
  • Person is ready to proceed with assessment

Referral to Application Progression

Typical Flow: Referral ‚Üí Application ‚Üí Case

  1. Referral Created: Initial contact, person inquiring
  2. Information Provided: Worker explains program, determines interest
  3. Application Created: Person decides to apply formally
  4. Assessment Conducted: Worker evaluates eligibility and need
  5. Case Created: If approved, active IHSS case established

Key Differences:

Aspect Referral Application
Formality Informal inquiry Formal request
Forms Required None SOC 873 Application
Processing Timeline No deadline Must process within 45 days
Documentation Minimal Extensive (medical, financial)
Outcome May or may not apply Approved or denied

Create Application

‚ñº

Application Prerequisites:

  • Person record exists or will be created
  • Person expresses intent to apply for IHSS
  • Application form (SOC 873) received or in progress
  • Basic eligibility criteria appear to be met

Application Information Required:

Category Information Needed
Personal Name, DOB, SSN, gender, language, living arrangement
Contact Address, phone, email, emergency contact
Eligibility Citizenship/immigration status, residency, Medi-Cal
Medical Disability/condition, medical providers, hospitalizations
Financial Income, assets, resources (for IHSS-specific eligibility)
Household Household members, potential providers

Application Date Significance:

Application Date: The date application is filed

Importance:

  • Determines when 45-day processing clock starts
  • Establishes potential benefit effective date
  • May allow retroactive payments if approved
  • Cannot be changed once set

45-Day Processing Requirement:

Federal and state law requires applications be processed within 45 days:

  • Day 1: Application date
  • Day 45: Decision must be made (approve or deny)
  • Extensions allowed only for good cause (e.g., applicant hospitalized)
  • Notice of Action must be sent

A case is created when application is APPROVED and recipient is eligible for services.

Case Creation Triggers:

  • New Application Approved: First-time IHSS applicant approved
  • Reapplication Approved: Previously terminated recipient reapplies and approved
  • Inter-County Transfer: Case transferring from another county

Case Information Established:

Program Type IHSS, PCSP, WPCS

Cases must be assigned to a caseworker/social worker:

  • Geographic Assignment: Based on recipient's address/region

5 CIN Clearance & MEDS Integration

‚ñº

Statewide Client Index (SCI)

‚ñº

What is SCI?

The Statewide Client Index (SCI) is California's master person index for public assistance programs. It assigns a unique Client Index Number (CIN) to each person in the state assistance system.

Purpose of CIN:

  • Unique Identifier: Each person gets ONE CIN across all programs
  • Prevent Duplicates: Ensures same person not in system multiple times
  • Cross-Program Coordination: Links person across CalWORKs, CalFresh, Medi-Cal, IHSS
  • Data Integrity: Maintains accurate person information statewide

CIN Format:

Example: 12345678A

  • 8 numeric digits
  • 1 alpha character (checksum)
  • Unique to each person
  • Never reused

CIN Clearance Process

‚ñº

What is CIN Clearance?

CIN Clearance is the process of searching SCI for an existing CIN or obtaining a new CIN when creating a case.

CIN Clearance Workflow:

Step 1: Worker Enters Person Information

When creating application or case, worker enters: name, DOB, SSN, gender

Step 2: CMIPS Queries SCI

CMIPS automatically sends real-time query to SCI with person information

Step 3: SCI Searches for Match

SCI searches for existing person record based on SSN, name, DOB

Step 4: SCI Returns Result

Three possible outcomes:

  • Exact Match Found: Returns existing CIN
  • Possible Matches Found: Returns list of potential matches
  • No Match Found: Creates new CIN

Step 5: Worker Reviews & Confirms

If possible matches, worker reviews to determine if any are same person

CIN Clearance Scenarios:

Scenario 1: Exact Match - CIN Found

SCI finds exact match based on SSN:

  • Existing CIN retrieved
  • CMIPS uses that CIN for the case
  • No worker action needed
  • Case creation continues automatically

Scenario 2: Possible Matches - Manual Review Required

SCI finds one or more possible matches:

  • Worker reviews each possible match
  • Compares name, DOB, SSN, address
  • Determines if any match is same person
  • If match: Select that CIN
  • If no match: Request new CIN

Scenario 3: No Match - New CIN Created

No matches found in SCI:

  • SCI creates brand new CIN
  • CMIPS stores new CIN with person record
  • Person now in SCI database
  • Case creation continues

CIN Data Mismatch:

If person information in CMIPS doesn't match SCI data:

  • Worker sees "CIN data does not match applicant data" screen
  • Worker reviews discrepancies (name spelling, DOB, etc.)
  • Worker determines which data is correct
  • Options:
    • Update CMIPS to match SCI
    • Request SCI update to match CMIPS
    • Create case without CIN (rare, requires justification)

MEDS Integration

‚ñº

What is MEDS?

MEDS (Medi-Cal Eligibility Data System) is California's system that stores Medi-Cal eligibility information.

Why IHSS Needs MEDS:

IHSS requires Medi-Cal eligibility as prerequisite:

  • ‚úì Recipient must have active Medi-Cal
  • ‚úì MEDS provides real-time eligibility status
  • ‚úì CMIPS verifies eligibility before authorization
  • ‚úì Daily updates ensure current status

MEDS Data Retrieved:

Data Element Description
Aid Code Type of Medi-Cal eligibility (e.g., SSI linked, Aged/Disabled)
Eligibility Status Active, terminated, suspended
Effective Dates Start and end dates of eligibility
Share of Cost Monthly SOC amount (if any)
County Code County administering Medi-Cal

MEDS Interface Types:

  • Real-Time Query: CMIPS queries MEDS when creating/viewing case
  • Daily Batch: Nightly file from MEDS with all eligibility updates
  • Monthly Renewal: Monthly file with SOC and aid code updates

No Active Medi-Cal

If MEDS shows no active Medi-Cal eligibility:

  • Cannot create IHSS case
  • Application must be pended or denied
  • Applicant referred to Medi-Cal eligibility worker
  • Once Medi-Cal active, IHSS application can proceed

SCI Daily Updates:

CMIPS receives daily batch file from SCI with updates:

  • CIN changes or corrections
  • Person demographic updates
  • Address changes
  • Death notifications

These updates automatically sync to CMIPS overnight.
