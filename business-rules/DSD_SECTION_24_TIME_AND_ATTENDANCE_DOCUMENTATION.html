1 Timesheet Processing Overview

‚ñº

Three Timesheet Methods

IHSS providers can submit timesheets using three different methods:

üìÑ Paper Timesheets (SOC 2261)

Traditional paper forms processed through TPF (Third Party Forms) mailroom with OCR/ICR scanning

üì± TTS - Telephone Timesheet System

Providers call toll-free number to enter hours via phone keypad

üíª ESP - Electronic Services Portal

Providers submit timesheets online through secure web portal

Timesheet Lifecycle

  1. Issuance: Timesheet issued to provider (paper, TTS, ESP)
  2. Submission: Provider completes and submits timesheet
  3. Receipt & Scanning: Paper forms scanned via TPF; electronic directly received
  4. Validation: Business rules validate against authorization and eligibility
  5. Exception Handling: Errors flagged for county review
  6. Approval: County reviews and approves/rejects
  7. Payroll Submission: Approved timesheets sent to payroll system
  8. Payment: Provider receives payment via direct deposit or check

Timesheet Status Workflow

Status Description
Issued Timesheet created and sent to provider
Received Timesheet submitted by provider, awaiting processing
In Process Being scanned, validated, or processed
Exception Validation errors found, requires county review
Pending Approval Awaiting supervisor approval (homemaker mode)
Approved Validated and approved, ready for payroll
Submitted to Payroll Sent to payroll system for payment processing
Paid Payment issued to provider
Rejected Timesheet rejected, provider must resubmit
Cancelled Timesheet cancelled, will not be paid

2 TPF Paper Timesheet Processing

‚ñº

Third Party Forms (TPF) vendor processes paper timesheets through mailroom, scanning, OCR/ICR recognition, and data completion.

TPF Mailroom Operations

‚ñº

Mailroom Process Flow:

Step 1: Receipt & Sorting

TPF mailroom receives paper timesheets from providers via mail. Forms sorted by type (timesheets vs. travel claims vs. sick leave).

Step 2: Date Stamp

Each form stamped with receipt date for tracking and audit purposes.

Step 3: Batching

Forms grouped into batches for processing. Batch tracking numbers assigned.

Step 4: Quality Check

Visual inspection for damaged, incomplete, or illegible forms.

Step 5: Forward to Scanning

Batches sent to image capture team for high-speed scanning.

Damaged or Illegible Forms

Forms that are torn, water-damaged, or illegible are flagged for special handling. County may need to contact provider for replacement timesheet.

Image Capture & Scanning

‚ñº

High-Speed Scanning Process:

  • Scanner Type: High-speed document scanners capable of 100+ pages per minute
  • Image Quality: 300 DPI (dots per inch) for clear OCR recognition
  • Color: Grayscale to capture all handwritten entries
  • Format: TIFF or PDF format for storage
  • Both Sides: Front and back of form captured

Image Storage:

Scanned images stored in secure document management system:

  • Indexed by timesheet number, provider ID, recipient ID
  • Retained for 7 years per records retention policy
  • Accessible to county staff via CMIPS
  • Available for audit, investigation, appeals

OCR/ICR Recognition

‚ñº

Optical Character Recognition (OCR):

Automated recognition of printed text and numbers on timesheet form:

  • Pre-Printed Fields: Timesheet number, recipient name, provider name, pay period
  • Checkboxes: Service type selections
  • Barcodes: 2D barcodes on form for identification
  • Accuracy: 95%+ for pre-printed fields

Intelligent Character Recognition (ICR):

Recognition of handwritten text (more challenging than OCR):

  • Hours Worked: Daily hour entries
  • Signatures: Provider and recipient signatures
  • Dates: Signature dates
  • Accuracy: 60-80% for handwritten fields
  • Learning System: Machine learning improves recognition over time

Recognition Confidence Levels

OCR/ICR system assigns confidence score to each recognized field:

  • High Confidence (90%+): Auto-accepted
  • Medium Confidence (70-89%): Flagged for review
  • Low Confidence (<70%): Requires manual data entry

TPF Data Completion

‚ñº

Manual Data Entry:

TPF staff manually complete fields that OCR/ICR could not recognize:

  • Review scanned image side-by-side with data fields
  • Manually type in hours, dates, signatures
  • Verify pre-populated fields are correct
  • Flag illegible entries for county follow-up

Quality Control:

  • Double-Entry Verification: Critical fields entered twice by different staff
  • Supervisory Review: Random sampling of completed timesheets
  • Error Rate Tracking: Monitor and report data entry accuracy
  • Training: Ongoing training to improve accuracy

Data Transmission to CMIPS:

Once data entry is complete:

  1. Timesheet data exported to secure file
  2. File transmitted to CMIPS via secure FTP
  3. CMIPS imports and validates data
  4. Timesheet appears in county's review queue

3 Electronic Timesheet Processing (TTS & ESP)

‚ñº

TTS - Telephone Timesheet System

‚ñº

TTS Overview:

Interactive Voice Response (IVR) system allows providers to submit timesheets via telephone.

Provider Registration:

  1. Provider requests TTS enrollment from county
  2. County generates TTS Registration Code
  3. Provider calls TTS toll-free number (1-866-376-7066)
  4. Provider enters registration code to enroll
  5. System assigns 4-digit PIN/passcode
  6. Provider can now submit timesheets via phone

TTS Timesheet Submission Process:

Step 1: Call TTS Number

Provider calls 1-866-376-7066 toll-free

Step 2: Authentication

Enter Provider ID and 4-digit PIN

Step 3: Select Recipient

If provider works for multiple recipients, select which one

Step 4: Enter Hours

Enter hours worked for each day of pay period using phone keypad

Step 5: Confirm Totals

System reads back total hours, provider confirms

Step 6: Submit

Provider submits timesheet; system provides confirmation number

TTS Benefits

  • No need to mail paper timesheets
  • Available 24/7
  • Immediate confirmation
  • No delays for postal mail
  • Faster payment processing

TTS Operations:

  • Register Provider for TTS: County enrolls provider
  • Reset TTS Passcode: If provider forgets PIN
  • Reset TTS Registration Code: Generate new registration code
  • View TTS Activity: Track TTS usage and submissions

ESP - Electronic Services Portal

‚ñº

ESP Overview:

Web-based portal where providers can submit timesheets, view payment history, update information, and communicate with county.

ESP Registration:

  1. Provider visits ESP website
  2. Clicks "Register" and enters personal information
  3. System verifies provider is active in CMIPS
  4. Provider creates username and password
  5. Email verification sent
  6. Provider logs in and can submit timesheets

ESP Timesheet Submission:

  1. Log In: Provider logs in to ESP
  2. Navigate to Timesheets: Select "Submit Timesheet"
  3. Select Recipient: Choose which recipient (if multiple)
  4. Select Pay Period: Choose semi-monthly pay period
  5. Enter Hours: Enter hours for each day
  6. Review: Review summary of hours and totals
  7. Submit: Electronically sign and submit
  8. Confirmation: Receive confirmation number and email

ESP Features:

  • Timesheet Submission: Submit regular timesheets
  • Payment History: View past payments and pay stubs
  • Update Information: Change address, phone, email
  • Direct Deposit: Enroll or update bank account
  • Tax Forms: View/print W-2, 1099 forms
  • Messages: Communicate with county
  • Benefits Enrollment: Enroll in health benefits (if eligible)
  • Recipient Signature: Recipient can co-sign timesheet electronically

ESP Recipient Signature

For Electronic Service Portal timesheets, recipient can be set up to co-sign electronically:

  • Recipient receives email notification when provider submits
  • Recipient logs in to ESP (or uses link in email)
  • Recipient reviews hours and electronically signs
  • Timesheet then released for processing

E-Timesheet Indicator

‚ñº

What is the E-Timesheet Indicator?

A flag in CMIPS that designates whether provider uses TTS/ESP or paper timesheets.

Setting the Indicator:

  • Automatic: Set automatically when provider enrolls in TTS or ESP
  • Manual: County can manually set if needed
  • Per Provider-Recipient Pair: Provider may use TTS for one recipient, paper for another

Impact of E-Timesheet Indicator:

Indicator Setting Timesheet Issuance
E-Timesheet = YES No paper timesheets mailed; provider uses TTS/ESP
E-Timesheet = NO Paper timesheets (SOC 2261) mailed to provider

Stop E-Timesheet Feature

If provider needs to temporarily switch back to paper timesheets:

  • County uses "Stop Provider E-Timesheet" function
  • E-Timesheet indicator set to NO
  • Paper timesheets will be issued for next pay period
  • Provider notified of change

4 Timesheet Validation & Business Rules

‚ñº

All timesheets (paper, TTS, ESP) go through extensive validation to ensure compliance with authorization, eligibility, and labor laws.

Validation Categories

‚ñº

1. Case & Provider Eligibility Validation:

  • ‚úì Recipient case is active and authorized
  • ‚úì Provider is active and assigned to recipient
  • ‚úì Provider enrollment status is approved
  • ‚úì No active provider ineligibility
  • ‚úì Case not on leave or terminated during pay period

2. Authorization Hours Validation:

  • ‚úì Hours do not exceed monthly authorized hours
  • ‚úì Hours do not exceed provider's assigned hours
  • ‚úì Service types match authorized services
  • ‚úì No hours claimed before authorization start date
  • ‚úì No hours claimed after authorization end date

3. MEDS POS (Point of Service) Eligibility:

  • ‚úì Recipient has Medi-Cal eligibility for service date
  • ‚úì No gap in Medi-Cal eligibility during pay period
  • ‚úì Proper aid code for service date
  • ‚úì Share of Cost status verified

4. Overtime Validation:

  • ‚úì Provider not exceeding 40 hours/week per recipient without overtime agreement
  • ‚úì Workweek definition exists for provider
  • ‚úì Overtime hours properly flagged and calculated
  • ‚úì Live-in provider overtime rules applied correctly

5. Signature & Certification Validation:

  • ‚úì Provider signature present (or electronic equivalent)
  • ‚úì Recipient signature present (or waived if appropriate)
  • ‚úì Signature dates within acceptable range
  • ‚úì Certification statement acknowledged

6. Travel Time Validation:

  • ‚úì Provider has authorized travel time between recipients
  • ‚úì Travel hours within authorized limits
  • ‚úì Travel between correct recipient pairs
  • ‚úì Travel dates align with work dates

Timesheet Exception Handling

‚ñº

When Validation Fails:

If timesheet fails any validation rule, it is placed in Exception Status and flagged for county review.

Exception Types:

Exception Type Description Action Required
Hours Over Authorization Hours exceed authorized amount Reduce hours or adjust authorization
MEDS POS Error No Medi-Cal eligibility for date Verify MEDS, may need to hold timesheet
Provider Not Active Provider terminated or not assigned Verify provider status, may reject timesheet
Case Terminated Case was terminated during pay period Prorate hours to termination date
Missing Signature Provider or recipient signature missing Contact provider/recipient for signature
Overtime Without Agreement Over 40 hrs/week, no agreement on file Reduce to 40 hrs or obtain agreement

County Exception Resolution:

  1. Review Exception: County staff reviews exception details
  2. Determine Cause: Identify reason for exception
  3. Take Action:
    • Approve timesheet as-is (if exception can be overridden)
    • Modify timesheet (adjust hours, dates)
    • Reject timesheet (provider must resubmit)
    • Cancel timesheet (will not be paid)
  4. Document Resolution: Add notes explaining action taken

Exception Task Assignment

When timesheet enters exception status, CMIPS creates a task assigned to appropriate county staff:

  • Case Owner: For case-related exceptions
  • Timesheet Reviewer: For routine exceptions
  • Supervisor: For high-dollar or complex exceptions

Share of Cost Deduction

‚ñº

SOC Deduction Process:

For recipients with Share of Cost, CMIPS calculates the SOC deduction when processing timesheets:

  1. Identify SOC Recipient: Check if recipient has MEDS SOC or IHSS SOC
  2. Calculate SOC Hours: Determine how many hours apply toward SOC
  3. Apply Deduction: Deduct SOC hours from payable hours
  4. Notify Provider: Show SOC deduction on payment stub

SOC Hour Calculation:

SOC hours are calculated based on county pay rate and provider's hours:

Formula: SOC Hours = (SOC Dollar Amount) / (County Pay Rate)

Example:

Recipient SOC: $360.00 per month

County Pay Rate: $18.00/hour

SOC Hours: $360 / $18 = 20 hours

Provider Worked: 80 hours

SOC Deduction: 20 hours

Payable Hours: 80 - 20 = 60 hours

5 Electronic Visit Verification (EVV)

‚ñº

Federal 21st Century Cures Act requires Electronic Visit Verification for personal care and home health services.

EVV Federal Requirement

As of January 1, 2021, states must implement EVV systems that capture:

  • Type of service performed
  • Individual receiving service
  • Date of service
  • Location of service delivery
  • Individual providing service
  • Time service begins and ends

EVV System Overview

‚ñº

How EVV Works:

Clock In - Service Start

Provider arrives at recipient's home and "clocks in" using:

  • Mobile App: Smartphone app with GPS
  • Fixed Device: Tablet or device in recipient's home
  • Telephone: Call EVV vendor's phone number
  • Web Portal: Log in via computer

Service Delivery

Provider delivers authorized IHSS services to recipient.

Clock Out - Service End

Provider "clocks out" when service is completed.

Data Transmission

EVV vendor transmits visit data to CMIPS:

  • Date and time of visit
  • Duration
  • GPS location (if mobile app)
  • Provider ID
  • Recipient ID

Timesheet Comparison

When timesheet submitted, CMIPS compares EVV data to timesheet hours:

  • ‚úì Match: Timesheet approved
  • ‚úó Mismatch: Flagged as exception for review

EVV Data Captured:

  • Provider ID: Who provided service
  • Recipient ID: Who received service
  • Check-In Time: Service start time
  • Check-Out Time: Service end time
  • Total Hours: Duration calculated
  • Location: GPS coordinates (if mobile)
  • Service Type: What services were provided

EVV Exception Timesheet Processing

‚ñº

When EVV Data Doesn't Match Timesheet:

If provider submits timesheet hours that don't match EVV recorded hours, an exception is created.

Common EVV Discrepancies:

Discrepancy Type Example Resolution
Hours Don't Match Timesheet: 4 hrs, EVV: 3.5 hrs Review, accept EVV or timesheet hours
Missing EVV Data Provider forgot to clock in/out Accept timesheet with attestation
Wrong Location GPS shows wrong address Verify actual location, override if appropriate
Extra Visits EVV shows visits not on timesheet Determine if valid, may add to timesheet

Good Cause Exceptions:

Federal regulations allow EVV exceptions for "good cause":

  • No Cellular Service: Rural areas with no cell coverage
  • Natural Disaster: Power outage, internet down
  • Technology Failure: EVV system malfunction
  • Safety Concerns: Domestic violence situations where GPS tracking raises safety concerns

Good Cause Documentation

When accepting timesheet despite EVV mismatch:

  • County must document good cause reason
  • Attestation from provider may be required
  • Pattern of exceptions may trigger investigation

EVV Vendor Integration

‚ñº

EVV Data Flow:

  1. Provider Clocks In/Out: Via EVV vendor's system
  2. EVV Vendor Stores Data: Visit data stored in vendor system
  3. Daily File Transmission: EVV vendor sends data to CMIPS daily
  4. CMIPS Import: EVV data imported and linked to case/provider
  5. Timesheet Comparison: When timesheet received, compared to EVV
  6. Exception Creation: Mismatches flagged for review

EVV Process Flows in CMIPS:

  • Assign New Case Provider EVV Flow: When provider assigned to case, register with EVV vendor
  • New Case EVV Flow: When new case authorized, set up EVV
  • Reactivate Case EVV Flow: When case reactivated from leave/termination
  • Rescind Termination EVV Flow: Restore EVV when termination rescinded
  • Case ICT EVV Flow: Inter-County Transfer handling

EVV Screens in CMIPS:

  • EVV Details: View EVV check-in/out details for timesheet
  • Print EVV Timesheet Confirmation: Print confirmation for provider
  • Print Recipient EVV Confirmation: Print confirmation for recipient

6 Travel Claim Processing

‚ñº

Providers who travel between multiple recipients may claim compensation for travel time using SOC 2255 Travel Claim Form.

Travel Time Eligibility

Travel time is compensable when provider travels DIRECTLY from Recipient A's home to Recipient B's home on the same workday. Travel from home to first recipient or from last recipient to home is NOT compensable.

Travel Claim Form Issuance

‚ñº

Travel Claim Issuance Process:

  • Automatic Issuance: Travel claim forms issued automatically to providers with authorized travel time
  • Manual Issuance: County can manually issue travel claim form
  • Replacement: Provider can request replacement if lost

Travel Claim Operations:

  • Travel Claim Issuance: Issue travel claim form to provider
  • Travel Claim Issuance - Replacement: Reissue lost form
  • View Travel Claim: Review submitted travel claim
  • Travel Claim History: Track all travel claims
  • Travel Claim Search: Find specific travel claims

Travel Claim Submission & Processing

‚ñº

Provider Completes Travel Claim:

  1. Fill Out Form: Provider enters travel hours for each day
  2. Recipient Pair: Indicates which two recipients traveled between
  3. Sign & Date: Provider signs and dates form
  4. Mail to County: Provider mails completed form (if paper)

TPF Travel Claim Processing:

Paper travel claims processed similarly to timesheets:

  • Mailroom receipt and sorting
  • Image capture and scanning
  • OCR/ICR recognition
  • Data completion
  • Transmission to CMIPS

Travel Claim Validation:

CMIPS validates travel claim against authorized travel time:

  • ‚úì Provider has active travel time authorization
  • ‚úì Recipient pair matches authorized travel route
  • ‚úì Hours within authorized travel time limits
  • ‚úì Dates align with work dates on timesheets
  • ‚úì No duplicate travel claims for same dates

Travel Claim Exceptions:

Exception Description
No Travel Authorization Provider doesn't have travel time approved
Wrong Recipient Pair Claimed travel between recipients not authorized
Hours Exceed Authorization Claimed more travel hours than authorized
Missing Timesheet Travel claim received but no corresponding timesheet

Travel Claim Received Before Timesheet

If travel claim is received before timesheet:

  • System holds travel claim in pending status
  • When timesheet received, travel claim processed together
  • If timesheet never received, travel claim may be rejected

7 Overtime & Payroll Processing

‚ñº

CMIPS calculates overtime in accordance with Fair Labor Standards Act (FLSA) and submits approved timesheets to payroll system.

Overtime Calculation

‚ñº

FLSA Overtime Rules:

Providers must be paid overtime (1.5x regular rate) for hours worked over 40 in their defined workweek.

Overtime Calculation Process:

  1. Identify Workweek: Use provider's workweek definition (e.g., Sunday-Saturday)
  2. Sum Weekly Hours: Total all hours worked in that workweek
  3. Regular Hours: First 40 hours at regular rate
  4. Overtime Hours: Hours over 40 at 1.5x rate
  5. Calculate Pay: (40 √ó regular rate) + (OT hours √ó 1.5 √ó regular rate)

Example:

Provider Workweek: Sunday through Saturday

Hours Worked:

  • Sunday: 0 hours
  • Monday: 8 hours
  • Tuesday: 10 hours
  • Wednesday: 8 hours
  • Thursday: 10 hours
  • Friday: 8 hours
  • Saturday: 2 hours

Total: 46 hours

Regular Hours: 40 at $18.00 = $720.00

Overtime Hours: 6 at $27.00 = $162.00

Total Pay: $882.00

Live-In Provider Overtime:

Special rules apply to providers who live in recipient's home:

  • Different Threshold: May have different overtime threshold (depends on state law)
  • Sleep Time: Sleep hours may not count toward regular hours
  • Meal Breaks: Meal break rules differ for live-in providers

Submit Timesheet to Payroll

‚ñº

Payroll Submission Process:

  1. Timesheet Approved: All validation passed, county approved
  2. Overtime Calculated: Regular and overtime hours computed
  3. SOC Deducted: Share of Cost deduction applied if applicable
  4. Tax Withholding: W-4/DE-4 elections applied
  5. Benefits Deduction: Health benefits premium deducted
  6. Payroll File Created: Data formatted for payroll system
  7. File Transmitted: Sent to state payroll system via secure interface

Payroll File Contents:

  • Provider ID and personal information
  • Recipient case number
  • Pay period dates
  • Regular hours and rate
  • Overtime hours and rate
  • Gross pay
  • Tax withholding elections
  • Deductions (SOC, benefits, etc.)
  • Net pay
  • Direct deposit information

Payment Timeline:

Event Typical Timeline
Pay Period Ends 15th or End of Month
Provider Submits Timesheet Within 7 days after pay period end
County Reviews & Approves 3-5 days
Submit to Payroll 1-2 days
Payroll Processing 2-3 days
Payment Issued 2-3 weeks after pay period ends

Direct Deposit vs. Paper Check

Direct Deposit: Payment deposited directly to provider's bank account (faster, more secure)

Paper Check: Check mailed to provider's address (slower, can be lost or stolen)

Counties strongly encourage direct deposit enrollment.

Payment Corrections

‚ñº

When Corrections Are Needed:

  • Timesheet was approved with incorrect hours
  • Provider was underpaid or overpaid
  • Wrong pay rate applied
  • SOC deduction calculated incorrectly
  • Overtime not properly calculated

Payment Correction Process:

  1. Identify Error: Provider or county discovers payment error
  2. Create Correction: County creates payment correction in CMIPS
  3. Type of Correction:
    • Supplemental Payment: If provider underpaid, issue additional payment
    • Overpayment Recovery: If provider overpaid, initiate recovery process
  4. Submit to Payroll: Correction submitted with next payroll cycle
  5. Notification: Provider notified of correction and amount

View Payment Correction List:

County staff can view all pending payment corrections to track status and ensure processing.
