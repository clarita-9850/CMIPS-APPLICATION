CMIPS

Technical Architecture

Executive Summary

CMIPS (Case Management Information & Payroll System) is an enterprise-grade platform supporting California's In-Home Supportive Services (IHSS) program across all 58 counties. Built with modern cloud-native technologies (Next.js 14, Spring Boot 3.2, Java 21, PostgreSQL 16, Apache Kafka 3.6, Keycloak 25). The platform implements a microservices-ready architecture with 6 deployed applications, 2 embedded frameworks, and comprehensive security through OAuth 2.0/OIDC authentication, role-based authorization, and field-level data masking.

System Components

Applications (6)

  • CMIPS Frontend - Next.js 14, React 19 / Node.js (Port 3000)
  • CMIPS Backend - Spring Boot 3.2 (Port 8080)
  • Scheduler Frontend - Next.js 14, React 19 / Node.js (Port 3002)
  • Scheduler Backend - Quartz (Port 8084)
  • API Gateway - Spring Cloud Gateway (Port 8090)
  • Keycloak - IAM (Port 8080)

Frameworks (2)

  • Integration Hub (BAW Framework) - SFTP/FTPS file processing
  • Test Governance Framework - Test coverage validation

Infrastructure (3)

  • PostgreSQL 16 - 4 databases
  • Apache Kafka 3.6 - 5 event topics
  • Redis 7.0 - Cache & Pub/Sub

Core Capabilities

  • Case Management: Recipient, provider, and case worker management workflows
  • Timesheet Processing: Submission, validation, and approval
  • Work Queue & Task Management: Automated assignment, prioritization, and tracking
  • Notification System: Real-time alerts and event-driven messaging
  • Batch Orchestration: Cron-based scheduling with DAG dependencies
  • External Integration: SFTP-based file exchange with state agencies (SCO, STO, DOJ, EDD)

Applications

1. CMIPS Frontend (Port 3000)

Technology: Next.js 14.2, React 19 / Node.js, TypeScript 5.3, TailwindCSS

Purpose: User interface for case workers, supervisors, and administrators across 58 counties

Key Features: Case management dashboards, timesheet submission/approval, provider management, real-time notifications, custom reporting

2. CMIPS Backend (Port 8080)

Technology: Spring Boot 3.2, Java 21 (Virtual Threads), Spring Batch 5.1, Spring Kafka 3.1

Purpose: Core business logic, REST APIs, and batch processing

Capabilities:

  • Case Management workflows
  • Timesheet processing with validation
  • Payment processing and payroll calculations
  • Work queue and task management
  • Notification system (real-time alerts, emails)
  • Authentication & authorization (JWT, RBAC)
  • Batch processing (payroll, file generation)
  • External system integration via Integration Hub
  • Reporting & analytics

Integration: REST APIs via API Gateway, Kafka event publishing (5 topics), Redis Pub/Sub for job events

3. Scheduler Frontend (Port 3002)

Technology: Next.js 14.2, React 19 / Node.js, React Flow (DAG visualization)

Purpose: Job orchestration monitoring and management UI

Key Features: Interactive DAG visualization, job execution monitoring, dependency management, calendar controls, real-time status updates

4. Scheduler Backend (Port 8084)

Technology: Spring Boot 3.2, Quartz Scheduler 2.3.2

Purpose: Enterprise job scheduling and orchestration with DAG-based dependency management

Features: Cron-based scheduling, DAG dependency resolution, job triggering via REST, event-driven execution, calendar integration, execution tracking with retry logic

Integration: Triggers CMIPS Backend batch jobs via REST; receives status updates via Redis Pub/Sub

5. Keycloak (Port 8080)

Technology: Keycloak 25

Purpose: Centralized identity and access management

Features: OAuth 2.0/OIDC authentication, JWT token issuance, SSO, user/group management, role-based access control, location-based authorization (58 county groups)

6. API Gateway (Port 8090)

Technology: Spring Cloud Gateway 4.1, Java 21

Purpose: Single entry point for all client requests with centralized security and routing

Features: Request routing (path-based), JWT validation, CORS handling, load balancing, security filters, circuit breaking

7. Test Governance Framework

Technology: Java 21, Maven Plugin, Java Reflection API

Purpose: Automated test coverage validation ensuring all business rules have corresponding test coverage

Features: @BusinessRule and @TestsRule annotations, build-time validation, coverage reporting (HTML/JSON), compliance mapping (HIPAA, FISMA, NIST, CMS), severity-based build failures, bidirectional traceability matrix

Deployment: Maven lifecycle (verify phase)

8. Integration Hub Framework (BAW)

Technology: Java 21, JSch (SFTP), Apache Commons IO/Lang3, Jackson (JSON/XML)

Purpose: File integration framework for batch processing. Supports SFTP and REST/HTTP as input/output destinations. CMIPS BAW uses SFTP for external agency integration.

Formats Supported: Fixed-width (PRDR120A, PRDR110A), CSV, JSON, XML, TSV, Pipe-delimited

External Systems (CMIPS BAW – SFTP only):

  • Outbound: SCO (payment requests)
  • Inbound: STO (warrant status), DOJ (background checks), EDD (employment verification)

Features: Annotation-driven schema definition, SFTP connection management, retry logic, format conversion, validation framework, optional REST/HTTP destinations

Deployment: Embedded in CMIPS Backend as Maven dependency

Security & Integration

Security Architecture (6 Layers)

  1. Network Security: HTTPS/TLS encryption for all client-server communication, secure SFTP for external file transfers
  2. API Gateway Security: JWT token validation, CORS policy enforcement, security headers
  3. Authentication: OAuth 2.0/OIDC via Keycloak, JWT tokens (RS256 algorithm), Single Sign-On (SSO)
  4. Authorization: Role-based access control (RBAC), location-based permissions (58 county groups), zero-hardcoding (all rules in Keycloak)
  5. Application Security: Input validation, SQL injection prevention, XSS protection
  6. Data Security: Field-level data masking, encryption at rest, comprehensive audit logging

Integration Patterns

Internal Integration

  • Frontend ↔ Backend: HTTPS/REST via API Gateway with JWT Bearer tokens
  • Scheduler ↔ CMIPS: REST API calls for job triggering + Redis Pub/Sub for status updates
  • Event Bus: Kafka for asynchronous event processing (5 topics: case, timesheet, provider, task, notification)
  • Cache Layer: Redis for session data and temporary storage
